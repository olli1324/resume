The four exam libraries (allowed on EXAM platform)

These cover most of the heavy lifting on the exam. Knowing which function to call is the actual skill being tested.

  • lib_fl: io_fl(f,g,h,'name') returns control law, state transformation, and relative degree automatically. lieder(f,h,m) for raw Lie derivatives.
  • lib_nmpc: nmpc_design_template.m is a ready scaffold. Fill in pred_model.m and nlcon.m, set (Ts, Tp, P, Q, R), call nmpc_design_st2(par).
  • lib_rotations: every conversion (axes2dcm, dcm2qua, qua2dcm, qua2euler, euler2qua), every operation (quatprod, quat_error, vec_rot_quat), kinematics (kin_quat, kin_tb321), Euler's equation (euler_eq).
  • lib_aerospace: oe2rv, rv2oe for orbital elements; spacecraft_dynamics for ode45; orb_control_blocks.slx for Simulink.

1

Diskret-tid rekursjon — regn $y(N)$

Ser ut som

«Consider the discrete-time system $y(k+1) = \ldots y(k) + \ldots u(k) + \ldots$. Compute $y(5)$ for input sequence $u = [\ldots]$ and initial conditions $y(1) = \ldots, y(2) = \ldots$.»

Fremgangsmåte
  1. Sett initialbetingelsene i en vektor y(1), y(2), ...
  2. For-løkke fra $k$ = nest siste kjente indeks opp til $N-1$.
  3. Les av $y(5)$.
y = zeros(N, 1);
y(1) = -4.5; y(2) = -3.5;
u = [-3.5 -1.75 -3.25 1.25];
for k = 2:N-1
    y(k+1) = y(k) + 2*y(k-1) - 2.5*u(k-1)^2 - 4*u(k-2);
end
y(5)
Felle

Svaralternativet «kan ikke beregnes fordi ulineært» er aldri riktig — numerikk fungerer på alle systemer. Indeks-felle: pass på om $u$ indekseres fra 1 eller 0, og om $u(k-1)$ er fortid eller fremtid.

2

Kontinuerlig-tid system ved tid $t$

Ser ut som

«Consider the system $\dot x_1 = \ldots, \dot x_2 = \ldots, y = \ldots$. Compute $y(t)$ for $t = 24$ s with initial state $x(0) = [\ldots]$.»

Fremgangsmåte
  1. Skriv $f(t, x)$ som anonym funksjon.
  2. Kjør ode45 over tidsintervallet.
  3. Les av siste verdi.
f = @(t,x) [5*sin(6*t) - x(1); 3*x(1)*x(2) - 2*x(2) + 1];
[T, X] = ode45(f, [0 24], [2; 1.5]);
y_end = X(end, 2);   % y = x2 i dette tilfellet
Felle

Samme som mønster 1 — «kan ikke beregnes» er feil. Default ode45-toleranser er nok.

3

Likevektspunkter

Ser ut som

«Find the equilibrium points of the system $\dot x = [u - 6 x_1 + x_2;\; x_1^2 - x_2 - 5]$ for constant input $\bar u = 6$.»

Fremgangsmåte
  1. Sett konstant input, løs $f(\bar x, \bar u) = 0$ med symbolic toolbox.
  2. Hver rad i xe er ett likevektspunkt. Sammenlign med svaralternativene.
syms x1 x2 real
u_bar = 6;
eq1 = u_bar - 6*x1 + x2 == 0;
eq2 = x1^2 - x2 - 5 == 0;
sol = solve([eq1, eq2], [x1, x2]);
xe = double([sol.x1, sol.x2])
Felle

Det kan finnes flere likevektspunkter (vanlig 2 for kvadratiske systemer). Sjekk alle.

4

Stabilitet via linærisering

Ser ut som

«Given the nonlinear system $\dot x = \ldots$, study the stability of the equilibrium point at $\bar u = 2$, $\bar x = [16.6; -95.58]$.»

Fremgangsmåte
  1. Beregn Jakobi-matrisen ved likevekt og finn egenverdiene.
  2. Les av egenverdiene:
    • Alle $\mathrm{Re}(\lambda) < 0$ → asymptotisk stabil
    • Minst én $\mathrm{Re}(\lambda) > 0$ → ustabil
    • $\mathrm{Re}(\lambda) = 0$ (resten $\leq 0$) → marginalt stabil (eller inkonklusiv for ulineær)
syms x1 x2 u real
f = [6*x1 - 2*u + x2; 2.5*x1^2 - x1 + 7*x2 - 3];
J = jacobian(f, [x1; x2]);
A = double(subs(J, [x1, x2, u], [16.6, -95.58, 2]))
eig(A)
Felle

«The system cannot be linearized» er nesten alltid feil. Du kan alltid linærisere.

5

Stabilitets-definisjoner (bookwork)

Ser ut som

«Let $\bar x$ be an equilibrium state of a nonlinear system. Choose the correct definition of simply stable.»

Fremgangsmåte — pugg dette
$\bar x$ is locally simply stable if, taking $x(0)$ sufficiently close to $\bar x$, $x(t)$ remains close to $\bar x$, $\forall t \geq 0$.

Distraktorer som er feil:

  • «if $x(0) = \bar x$, then $x(t) = \bar x$» — trivielt, sier ingenting.
  • «if $x(0)$ in an arbitrary neighbourhood» — feil, må være «sufficiently close».
TypeDefinisjon
Stable (Lyapunov)$x(0)$ nær → $x(t)$ nær
AsymptotiskStable + $x(t) \to \bar x$
Eksponensielt$\lVert x(t) - \bar x\rVert \leq c\, e^{-\lambda t} \lVert x(0) - \bar x \rVert$
LokalDet over, for $x(0)$ i en omegn
GlobalDet over, for alle $x(0)$
BIBOBegrenset input → begrenset output
6

Fase-portrett identifikasjon

Ser ut som

«Indicate which figure shows the trajectories of $\dot x = [\ldots; \ldots]$.» Fire bilder gitt.

Fremgangsmåte
  1. Finn likevektspunktet ved å sette $\dot x = 0$.
  2. Klassifiser linæriseringen ved likevekt (egenverdier av Jakobi):
    • Komplekse med $\mathrm{Re} < 0$ → stabil spiral
    • Komplekse med $\mathrm{Re} > 0$ → ustabil spiral
    • Reelle med samme tegn → stabil/ustabil node
    • Reelle med motsatte tegn → saddel
  3. Match med tegningene: spiraler har sirkulær flow, noder har rette innstrøminger, sadler har korsstruktur.
Snarvei — simuler raskt 
f = @(t,x) [-3*x(1) - x(2); 4*x(1)^2 - x(2) - 5];
for k = 1:5
    [~, X] = ode45(f, [0 5], 10*rand(2,1));
    plot(X(:,1), X(:,2)); hold on
end

7

Relativ grad

Ser ut som

«Consider $\dot x = f(x) + g(x) u$, $y = h(x)$ where $f(x) = [\ldots]$, $g(x) = [\ldots]$, $h(x) = x_4$. Determine the relative degree $\gamma$ of the system.»

Fremgangsmåte — bruk lib_fl direkte 
syms x1 x2 x3 x4 u real
x = [x1; x2; x3; x4];
f = [x1^2 + 2*x4; x3^2; x1*x4 - 0.5*x3; x2 + 0.2*x4];
g = [-1; 0; 0; 0];
h = x4;
[~, ~, gamma] = io_fl(f, g, h, 'sys');
disp(gamma)
Manuell sjekk

Deriver $h$ langs $f$ til input dukker opp. $\dot y = L_f h = f_4 = x_2 + 0.2 x_4$ (ingen $u$); $\ddot y = L_f^2 h = \ldots$ (sjekk om $u$ dukker opp). Første gang $L_g L_f^{r-1} h \neq 0$ er $\gamma = r$.

8

Feedback linearization kontroll-lov

Ser ut som

Samme system som mønster 7, men nå: «Derive the control law for feedback linearization.»

Fremgangsmåte — samme io_fl-kall, returnerer $u$ direkte 
[u_law, mu, gamma, a, b] = io_fl(f, g, h, 'sys');
disp(u_law)

Resultatet på formen $u = (-a + v) / b$ der $v$ er det nye lineariserende inputtet. Forenkle med simplify(u_law) og sammenlign termer mot svaralternativene.

9

Sliding mode controller-design

Ser ut som

«Design a sliding mode controller for $\dot x = \ldots$ to regulate $y \to 0$ with $\eta = 1$. Choose $(K_1, K_2)$ that minimizes $J = t_c + 0.2 \lVert u \rVert$.»

Fremgangsmåte
  1. Definer sliding surface $s(x)$ (vanligvis $s = x_2 + \lambda x_1$ for 2. orden).
  2. For hvert svaralternativ $(K_1, K_2)$: bygg Simulink med plant + sliding mode kontroller u = -K1*x1 - K2*x2 - eta*tanh(s), kjør med ode4, $T_s = 0.05$, $0 \leq t \leq 10$ s. Mål $t_c$ (når $|y| < 0.001$) og $\lVert u \rVert_2$, regn $J = t_c + 0.2 \lVert u \rVert$.
  3. Velg det med minste $J$.

Vanlig: 4 svaralternativer der 1–2 er ubrukbare og du tester de andre.

NMPC orbital eller rover

Ser ut som

«A satellite is traveling on a circular orbit. Design an NMPC algorithm to transfer to a target orbit. Choose $(T_s, T_p, R, P)$ that achieves tracking error $< \epsilon$.»

Fremgangsmåte — bruk lib_nmpc-templates 
% 1. Fyll inn pred_model.m med satellite-dynamikken (fra LS06)
% 2. Fyll inn nlcon.m hvis det er constraints
% 3. I nmpc_design_template.m, for hvert kandidat:
par.Ts = 20; par.Tp = 90;
par.R = diag([0.05 0.05 0.05]);
par.P = diag([0.4 7e5 7e5 7e5 7e5]);
par.Q = zeros(...);   % gitt
par.lb = -132*[1;1;1]; par.ub = 132*[1;1;1];
K = nmpc_design_st2(par);

Simuler i Simulink med nmpc_block_2025b.slx, mål tracking error mot toleranse, velg kandidaten som passer.

Felle

Et alternativ er ofte «kan ikke nås på grunn av propellant insufficiency». Sjekk om initial propellant er nok.

11

Quaternion PD attitude control

Ser ut som

«A satellite has inertia matrix $J = \mathrm{diag}(\ldots)$. Initial $q(0) = (1,0,0,0)$, $\omega(0) = 0$. Design $u = k_1 \tilde q - k_2 \omega$ to reach $q_f$. Convergence time $t_c < T$, command $\lVert u \rVert_\infty < U$. Choose $(k_1, k_2)$.»

Fremgangsmåte
  1. Bygg Simulink-modell:
    • Quaternion error: q_e = quatprod(quat_conj(q_ref), q)
    • Kontrolllov: u = k1*q_e(2:4) - k2*omega
    • Dynamikk: euler_eq(omega, J, u) integrert
    • Kinematikk: kin_quat(q, omega) integrert
  2. For hvert kandidat-par $(k_1, k_2)$: simuler, mål $t_c$ (når $\lVert \tilde q\rVert_\infty < 0.005$) og $\max \lVert u \rVert_\infty$.
  3. Velg det som oppfyller begge spec.

Snarvei: bruk att_kin_dyn_2025a.slx fra lib_rotations som ferdig oppsett.

EKF-design

Ser ut som

«Consider SISO nonlinear system $\dot x = f(x, u) + w$, $y = h(x) + v$. Design discrete-time EKF with $T_s = 0.1$ s. Choose $(Q^d, R^d)$ minimizing RMS $= \lVert x - \hat x\rVert / \sqrt{L}$.»

Fremgangsmåte
  1. Bygg Simulink-modell med plant + støy ($d_u$, $d_y$ som Random Number-blokker) og EKF-blokk:
    • Predict: $\hat x_{k+1|k} = \hat x_k + T_s f(\hat x_k, u_k)$
    • $F_k = I + T_s \frac{\partial f}{\partial x}\big|_{\hat x_k}$
    • $P_{k+1|k} = F_k P_k F_k^\top + Q^d$
    • $K = P_{k+1|k} H^\top (H P_{k+1|k} H^\top + R^d)^{-1}$
    • Update: $\hat x_{k+1} = \hat x_{k+1|k} + K(y - h(\hat x_{k+1|k}))$
  2. Bruk ode4, $T_s = 0.05$ s, sinusoidal $u$ med spec-amplitude, 60 s sim.
  3. For hvert kandidat-par $(Q^d, R^d)$: kjør, regn RMS, sammenlign.
Felle

«Estimated state diverges» er ofte et av svaralternativene — sjekk om EKF holder seg endelig.

Vektor-transformasjon mellom rammer

Ser ut som

«F1 is rotated wrt F2 according to a 221 rotation with angles $(\phi, \theta, \psi) = (\ldots)$. A vector in F1 is $v_1 = (\ldots)$. Find $v_2$.»

Fremgangsmåte — bruk axes2dcm 
phi = 2.1553; theta = 2.3531; psi = 3.0489;
T = axes2dcm([2 2 1], [phi theta psi]);   % F2 fra F1
v1 = [-0.6; -2; -5.3];
v2 = T' * v1   % invers, fordi T tar F1 -> F2
Felle

Konvensjons-spørsmål. Hvis oppgaven sier $I_k = T_{221} i_k$, går $T$ fra F2-akser til F1-akser. Vektor-transformasjon er motsatt: $v_2 = T \cdot v_1$ eller $T^\top \cdot v_1$ avhengig av definisjonen. Test begge mot svaralternativene.

Kvaternion fra rotasjons-sekvens

Ser ut som

«Consider a 132 rotation with angles $(\phi, \theta, \psi) = (\ldots)$. Compute the corresponding quaternion.»

Fremgangsmåte — bruk axes2qua 
q = axes2qua([1 3 2], [0.3927 3.1416 1.5708])

Eller bygg manuelt fra elementære kvaternioner:

q1 = ele_quat(0.3927, 1);   % aksen 1
q2 = ele_quat(3.1416, 3);   % aksen 3
q3 = ele_quat(1.5708, 2);   % aksen 2
q = quatprod(quatprod(q1, q2), q3)
Felle

Rekkefølge betyr alt. «132» betyr først om akse 1, så akse 3, så akse 2.

Akse av rotasjon fra DCM

Ser ut som

«Compute the axis of rotation of $T = [\ldots]$.»

Fremgangsmåte — Eulers teorem 
T = [0.99944 0.033412 0; -0.03324 0.99429 -0.10144; ...];
cos_theta = (trace(T) - 1) / 2;
theta = acos(cos_theta);
% Aksen er egenvektoren til T med egenverdi 1
[V, D] = eig(T);
[~, idx] = min(abs(diag(D) - 1));
axis = real(V(:, idx))

Alternativt direkte via qua2axes(dcm2qua(T)).

Kinematic equation integration

Ser ut som

«Consider kinematic equations for a 321 rotation. Initial $(\phi(0), \theta(0), \psi(0)) = (\ldots)$, input $\omega = (\ldots) \sin(t)$. Compute angles at $t = 24.8$ s.»

Fremgangsmåte — integrer kinematikk-ligningen 
phi0 = 0.31156; theta0 = 0.23895; psi0 = 0.2768;
omega_amp = [0.1621; -0.40329; -0.47023];
f = @(t, ang) kin_tb321(ang, omega_amp * sin(t));
[T, ANG] = ode45(f, [0 24.8], [phi0; theta0; psi0]);

Eller bruk kvaternion-versjonen for stabilitet:

q0 = euler2qua([phi0 theta0 psi0], [3 2 1]);
f = @(t, q) kin_quat(q, omega_amp * sin(t));
[T, Q] = ode45(f, [0 24.8], q0);
final_euler = qua2euler(Q(end,:)', [3 2 1])

17

True anomaly fra perigee-tilstand

Ser ut som

«Satellite mass $14164$ kg, at perigee $r_p = (6380434, 0, 0)$ m, $v_p = (0, 9692.8824, 0)$ m/s. Determine true anomaly $\theta$ after $0.7$ hours.»

Fremgangsmåte
  1. Sjekk om orbit er lukket (energi): 
    mu = 3.986e14;   % Earth's gravitational parameter
r0 = [6380434; 0; 0];
v0 = [0; 9692.8824; 0];
energy = norm(v0)^2/2 - mu/norm(r0);
% Hvis energy >= 0 -> ikke lukket, true anomaly udefinert
  2. Hvis lukket: integrer 2-legeme og finn vinkelen mellom $r_\text{perigee}$ og $r_\text{end}$: 
    f = @(t, x) [x(4:6); -mu/norm(x(1:3))^3 * x(1:3)];
[T, X] = ode45(f, [0 0.7*3600], [r0; v0], odeset('RelTol',1e-10));
r_end = X(end, 1:3)';
theta = acos(dot(r0, r_end) / (norm(r0) * norm(r_end))) * 180/pi
  3. Sammenlign med svaralternativene.
Felle

Hvis hastigheten gir hyperbolsk orbit (energi $> 0$), er riktig svar «true anomaly cannot be determined». Sjekk alltid energien først.

Generell eksamen-flyt — for hvert spørsmål
  1. Identifiser mønstret (10–15 sek) — se på inputtype og hva som spørres etter.
  2. Åpne riktig template/funksjon (30 sek) — vit hvor io_fl og nmpc_design_template.m ligger.
  3. Fyll inn problemet (2–3 min) — symboler, parametre, initialbetingelser.
  4. Kjør (1–2 min) — vent på simulering.
  5. Sammenlign mot svaralternativer (30 sek) — velg den nærmeste.

Total: 5–8 min per gjenkjent spørsmål. For 11 spørsmål: 1–1,5 t aktiv jobbing, 30–45 min sjekk. Gjenkjenner du ikke et mønster: hopp og kom tilbake — negativ score betyr ikke alltid prøv.
